# STRUCTURE

## Repository Layout
- Current workspace root contains:
- `.codex/` (GSD tooling and workflows)
- `marketplace_backend/` (Rails API application)
- `.planning/` (planning artifacts generated by GSD workflows)

## Rails App Root
- `marketplace_backend/README.md`
- `marketplace_backend/Gemfile`
- `marketplace_backend/Gemfile.lock`
- `marketplace_backend/Rakefile`
- `marketplace_backend/config.ru`

## Application Code Directories
- Controllers: `marketplace_backend/app/controllers/`
- Models: `marketplace_backend/app/models/`
- Jobs: `marketplace_backend/app/jobs/`
- Mailers: `marketplace_backend/app/mailers/`
- Views (mailer templates): `marketplace_backend/app/views/layouts/`

## Configuration Directories
- Core config: `marketplace_backend/config/`
- Environments: `marketplace_backend/config/environments/`
- Initializers: `marketplace_backend/config/initializers/`
- Locales: `marketplace_backend/config/locales/`

## Data and Runtime Directories
- DB files/seeds: `marketplace_backend/db/`
- Logs: `marketplace_backend/log/`
- Temp files/PIDs: `marketplace_backend/tmp/`
- Persistent storage placeholder: `marketplace_backend/storage/`

## Testing Layout
- Test bootstrap: `marketplace_backend/test/test_helper.rb`
- Integration tests: `marketplace_backend/test/integration/`
- Current custom test: `marketplace_backend/test/integration/healthcheck_test.rb`

## Naming Conventions in Use
- Standard Rails naming and directory conventions.
- Environment-separated config files.
- `Application*` base classes for framework extension.
